# Sprint 4.5 — Entity Grounding + Defaults Flip + UI Copy Scrub (Design Spec)

**Goal.** Stop cross-entity false-positive pairs from forming at the aligner by detecting equipment IDs per page (LLM) and binding extracted parameter values to them. Flip all v2 Track 2 defaults ON so the deployed Streamlit experience showcases the full AI stack on first load. Scrub internal jargon (`Track`, `Sprint`, `v1.5`, `Phase`) from every reviewer-facing string.

**Exit tag:** `v2.4-grounding`. **PIVOT_PLAN reference:** "BACKLOG R-F entity fingerprinting" + Sprint 5 prerequisite.

---

## §1 Approach + Components

Three changes ship together as Sprint 4.5.

### (a) Entity grounding

One Claude Sonnet 4.5 call per page returns the equipment-ID inventory `[(label, kind, y_top, y_bottom), ...]`. Diskcached on `(PDF content hash, page, prompt_version, model)`. Parallel-per-page via `ThreadPoolExecutor(5)`.

Track 1 + Track 2 records bind to detected entities by **y-range enclosure with nearest-y fallback**: each record's `bbox_y_center` is tested against every same-page entity's `[y_top, y_bottom]` range; on multiple enclosures pick the tightest fit; on no enclosure fall back to nearest by y-distance.

Phase 19's existing same-entity-only rule in `align_exact._filtered_pool` then refuses cross-entity matches automatically. No alignment code changes.

## §2 Schema + Entity Detector Logic

### Schema (`src/interlock/llm_pipeline/schemas/entity.py`)

```python
from typing import Literal
from pydantic import BaseModel, ConfigDict, Field


class DetectedEntity(BaseModel):
    model_config = ConfigDict(frozen=True)
    label: str = Field(min_length=1, max_length=128)
    kind: Literal["equipment", "circuit", "section", "unknown"]
    y_top: float = Field(ge=0.0)
    y_bottom: float = Field(ge=0.0)
    page: int = Field(ge=1)


class PageEntities(BaseModel):
    model_config = ConfigDict(frozen=True)
    page: int = Field(ge=1)
    entities: list[DetectedEntity]
```

Caller validates `y_top <= y_bottom` (pydantic doesn't cross-check fields cheaply); violations dropped by the detector's sanity guard.

### Detector (`src/interlock/llm_pipeline/entity_detect.py`)

```python
def detect_entities_on_page(pdf_path: str, page: int) -> list[DetectedEntity]:
    """Return equipment/circuit/section entities on one page.

    Diskcache namespace 'llm-entities' keyed by (PDF content hash, page,
    PROMPT_VERSION, model). Cold ~$0.005/page; warm $0. On API / parse /
    validation failure → returns [] (caller treats as 'no entities
    detected on this page', graceful fallback).
    """

def detect_entities_for_doc(
    pdf_path: str, max_workers: int = 5,
) -> dict[int, list[DetectedEntity]]:
    """Parallel-per-page wrapper. Returns {page_number: [DetectedEntity, ...]}."""
```

**Per-page flow:**

1. Render page text via PyMuPDF + capture page bbox dimensions.
2. Build prompt: page text + page height + the kinds prompt should distinguish.
3. Call Sonnet 4.5 with structured JSON request.
4. Pydantic validate response → `list[DetectedEntity]`.
5. **Sanity guard**: drop entities where `y_bottom < y_top`; drop entities whose `label` is on the stoplist `{"IEEE", "IEC", "NEMA", "ANSI", "UL", "Example", "Figure", "Note", "Table", "See"}`.
6. Failure → return `[]` (treat page as having no detectable entities; binding will fall back to leaving records' entity_tag empty).

### Binding helper (`src/interlock/extract/entity_bind.py`)

```python
def bind_records_to_entities(
    records: list[ParameterRecord],
    entities_by_page: dict[int, list[DetectedEntity]],
) -> list[ParameterRecord]:
    """Return new records with entity_tag populated by y-binding.

    For each record, on its page: find entity whose [y_top, y_bottom]
    encloses the record's bbox y-center. If multiple enclose, pick the
    one with smallest y-range (tightest fit). If none enclose, fall back
    to nearest entity by y-distance (only if any same-page entity exists).
    Preserves existing entity_tag if already non-empty (Track 1 leading-row
    marker wins, since it's a stronger signal than spatial proximity)."""
```

### Prompt (`src/interlock/llm_pipeline/prompts/entity_detect.md`)

Engineering-aware extractor with explicit `kind` classification rules:
- **equipment** — physical hardware: `XFMR-NNN`, `M-NNN`, `P-NNN`, `F-NNN`, `JCN…`, etc.
- **circuit** — labelled feeder/bus/branch: `200A Feeder`, `Branch 1`, `Bus A`, `Main`.
- **section** — free-text headings: `Example 1`, `Step 2`, `Coordination Study`.
- **unknown** — anything not confidently one of the above.

Explicit NOT-to-extract list: standards bodies (IEEE/IEC/NEMA/ANSI/UL), figure/table references, page headers.

### Cost ledger

Every call appends a `cost_event` row with `namespace='llm-entities'`, `model='claude-sonnet-4-5'`, token counts from response usage. Same Phase 13 pattern.


### Stage callback

New stage id `entity_detect` between `extract` and `align`. UI maps to `Detecting equipment IDs (AI)`. Skipped when toggle off.

### No alignment-code changes

Phase 19's existing same-entity rule in `align_exact._filtered_pool` is already strict-same-entity. Once both records have tags, cross-entity pairing refuses by construction. The 200A/400A Feeder and 77A/42A Motor FLA false positives both die at this gate.

### Asymmetric-detection edge case

If detector finds an entity on Doc A's page but not Doc B's same page (model variance), Doc A's records get a tag and Doc B's don't. Same-entity rule then refuses to pair them — they show up in `unpaired_a` / `unpaired_b` honestly instead of forming a misleading pair. Honest gap > false positive.

### Snapshot-equivalence tests

Every existing test that asserts `v1.5-mvp-ready` invariants explicitly passes the v1.5 toggle set:

```python
result = review_two_documents_full(
    DOC_A, DOC_B, embed_fn=_trivial_embedder,
    classify_docs=False,
    use_llm_extraction=False,
    use_llm_reranker=False,
    use_entity_grounding=False,
)
```

So the deterministic floor stays bit-identical; just no longer the default. The 354-test v2.2 invariant suite stays green after the flip.

### Failure modes (all caught)

- Anthropic API outage → detector returns `{}`; records keep existing tags.
- Pydantic validation error on one page → that page yields `[]`; other pages continue.
- Stoplist filter drops everything → page yields `[]`.
- PDF render failure on a page → that page yields `[]`.

### Per-flag header chip helper

```python
def _entity_chip(flag: Any) -> str:
    """Return entity-tag chip text for the flag header.

    Same-tag both sides → ' · 🏷️ <tag>' (one chip).
    Different tags → ' · 🏷️ A:<a_tag> / B:<b_tag>' (rare: detector
    asymmetry; also a weak-pair signal the reranker would have caught).
    Both empty → '' (silent — most flags pre-grounding).
    """
    a_tag = (getattr(flag.a_record, "entity_tag", "") or "").strip()
    b_tag = (getattr(flag.b_record, "entity_tag", "") or "").strip()
    if not a_tag and not b_tag:
        return ""
    if a_tag and b_tag and a_tag == b_tag:
        return f" · 🏷️ {a_tag}"
    if a_tag and b_tag:
        return f" · 🏷️ A:{a_tag} / B:{b_tag}"
    return f" · 🏷️ {a_tag or b_tag}"
```

Header construction appends `ent_chip` after `prov_badge`:

## §7 Risks + Mitigations

| Risk | Mitigation |
|---|---|
| Detector hallucinates entities not on page | Stoplist + pydantic validation; downstream binding uses y-coords which only resolve when the entity is real. |
| Asymmetric detection (A has tag, B doesn't) → real mismatch lost as unpaired | Surfaces in `unpaired_a/b` honestly. Documented in TDD known-limits. |
| VC demo fixture happens to be detector's weakness | Live-API exit gate gates sprint exit on the locked fixture. |
| Cost surprise on big PDFs | Cold ~$0.005/page; cached after first run. Sidebar help states cost explicitly. |
| Default-on flip regresses v1.5 invariant suite | Every existing snapshot test rewritten to pass `False` explicitly. CI gates the deterministic floor. |
| Per-flag chip pollutes UI when most flags tagged identically | Single chip when both match. Differs only when detector disagrees across docs — rare + meaningful signal. |
| Live exit-gate flake (LLM nondeterminism) | Detector outputs cached per (PDF hash, page); the first cold run sets the cache, subsequent runs deterministic. Re-run on flake. |
